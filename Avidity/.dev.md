## Directory

| path                        | description |
| :-------------------------- | :---------- |
| `Assets/Application/`       | Application frontend
| `Assets/Application/Execs/` | Application managers
| `Assets/Application/Parts/` | Reusable components
| `Assets/Application/Views/` | Distinct portions of the interface â€“ navbar, selector, etc.
|                             |
| `Assets/Modules/`           | Application backend modules
| `Assets/Modules/Core/`      | `static/` classes for frontend to interface with |
| `Assets/Modules/Bases/`     | Base classes
| `Assets/Modules/Types/`     | Data structures. These should only encapsulate data, *not* behaviour! |
| `Assets/Modules/Utils/`     | Utility
| `Assets/Modules/Internal/`  | Messy internals
| `Assets/Modules/External/`  | External modules

## Naming Conventions

| context        | casing |
| :------------- | :----- |
| folders        | `PascalCase` |
| files          | `kebab-case` |
| public fields  | `camelCase`  |
| private fields | `snake_case` |
| GameObjects    | `Title Case` |
